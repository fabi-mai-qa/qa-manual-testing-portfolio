# TC-004.07 - Get Order using a valid `orderId` 

## Description  
Validate successful retrieval of an order by a valid `orderId`.

## Preconditions  
- User is authenticated with a valid API token.
- A valid order was previously created successfully.
- A valid `orderId` is available.

## Endpoint
/orders/{orderId}

## Headers
Authorization: Bearer <valid_token>

## Payload
*Not applicable*

**Expected Result:**  
- Order is retrieved successfully.
- HTTP Status Code: 200 OK.
- Response contains:
	- Correct `orderId`
	- Correct `userId`
	- All order items created in the POST request
	- Correct seller and shipping information per seller
	- Correct `totalPrice`
	- Date and time of order creation in `createdAt`
- Returned data matches the order created previously

**Business Rules Covered:**
- Orders can be retrieved by a valid `orderId`.
- Order data integrity is preserved after creation.
- The system returns complete order information.
 
**Status:**  
- Not Executed / Passed / Failed

**Comments:**  
- Order retrieved successfully using GET after creation via POST.
