# TC-004.08 - Get Order using a non-existing `orderId` for the authenticated user

## Description  
Validate that the system returns an appropriate error response when attempting to retrieve an order using a non-existing `orderId` for the respective user.

## Preconditions  
- User is authenticated with a valid API token.
- Orders API is available.
- A non-existing `orderId` or an existing `orderId` that belongs to a different user, both in a valid format.

## Endpoint
/orders/{orderId}

## Headers
Authorization: Bearer <valid_token>

## Payload
*Not applicable*

**Expected Result:**  
- Order is not retrieved.
- HTTP Status Code: 404 Not Found.
- Response body returns a clear error message indicating that the order was not found.
- No order data is returned.

**Business Rules Covered:**
- Orders must be retrieved only when a valid and existing `orderId` is provided.
- The system must return 404 Not Found when the requested order does not exist or does not belong to the authenticated user.
 
**Status:**  
- Not Executed / Passed / Failed

**Comments:**  
- For data safety purposes, even if the provided `orderId` exists in the system but belongs to another user, the API must still return 404 Not Found with the generic error message that the order was not found.
