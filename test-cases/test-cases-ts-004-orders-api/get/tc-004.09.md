# TC-004.09 - Get Order using an invalid `orderId` format

## Description  
Validate that the system returns an appropriate error response when attempting to retrieve an order using an invalid `orderId` format.

## Preconditions  
- User is authenticated with a valid API token.
- Orders API is available.
- An `orderId` with an invalid format (e.g. alphanumeric string, special characters or empty value).

## Endpoint
/orders/{orderId}

## Headers
Authorization: Bearer <valid_token>

## Payload
*Not applicable*

**Expected Result:**  
- Order is not retrieved.
- HTTP Status Code: 400 Bad Request.
- Response body returns a clear error message indicating that the `orderId` format is invalid.
- No order data is returned.

**Business Rules Covered:**
- Orders must be retrieved only when a valid `orderId` format is provided.
- The system must return 400 Bad Request when the provided `orderId` has an invalid format.
 
**Status:**  
- Not Executed / Passed / Failed

**Comments:**  
- Invalid input format is correctly rejected before any business validation occurs.
