# TC-004.10 - Get Order without authentication token

## Description  
Validate that the system returns an appropriate error response when attempting to retrieve an order without an authentication token.

## Preconditions  
- User is not authenticated (no authentication token is provided).
- Orders API is available.
- A valid `orderId` exists in the system.

## Endpoint
/orders/{orderId}

## Headers
*No authentication token is provided*

## Payload
*Not applicable*

**Expected Result:**  
- Order is not retrieved.
- HTTP Status Code: 401 Unauthorized.
- Response body returns a clear error message indicating that authentication is required.
- No order data is returned.

**Business Rules Covered:**
- Orders must be retrieved only when the user is authenticated with a valid API token.
- The system must return 401 Unauthorized when authentication credentials are missing.
 
**Status:**  
- Not Executed / Passed / Failed

**Comments:**  
- Access to protected resources is correctly blocked when no authentication token is provided.
