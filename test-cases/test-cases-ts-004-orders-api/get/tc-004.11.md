# TC-004.11 - Get Order with expired/invalid token

## Description  
Validate that the system returns an appropriate error response when attempting to retrieve an order with an expired or invalid authentication token.

## Preconditions  
- User provides an expired or invalid token.
- Orders API is available.
- A valid `orderId` exists in the system.

## Endpoint
/orders/{orderId}

## Headers
Authorization: Bearer <expired_or_invalid_token>

## Payload
*Not applicable*

**Expected Result:**  
- Order is not retrieved.
- HTTP Status Code: 401 Unauthorized.
- Response body returns a clear error message indicating that the authentication token is invalid.
- No order data is returned.

**Business Rules Covered:**
- Orders must be retrieved only when the user is authenticated with a valid API token.
- The system must return 401 Unauthorized when authentication credentials are expired or invalid.
 
**Status:**  
- Not Executed / Passed / Failed

**Comments:**  
- Access to protected resources is correctly blocked when expired or invalid authentication token is provided.
