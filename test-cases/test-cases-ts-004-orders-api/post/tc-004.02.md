# TC-004.02 - Create Order with `hasShippingFee = false` and shipping fee value ignored

## Description  
Validate successful order creation when a shipping fee value is sent in the request but `hasShippingFee` is set to false.

## Preconditions  
- User is authenticated with a valid API token.
- Seller exists with valid shipping configurations.
- Products exist and are linked to the respective seller.

## Endpoint
/orders   

## Payload
```json
{
  "userId": 123,
  "orderItems": [
    {
      "productId": 10,
      "quantity": 2,
      "unitPrice": 50.00,
      "sellerId": 145
    }
  ],
  "shippingBySeller": [
    {
      "sellerId": 145,
      "hasShippingFee": false,
      "shippingFeeValue": 10.00,
      "shippingType": "STANDARD",
      "estimatedDeliveryDays": 7
    }
  ]
}
```

**Expected Result:**  
- Order is created successfully.
- HTTP Status Code: 201 Created.
- Shipping fee is not applied.
- Total price is calculated as:
	```text
	 2 × 50.00 = 100.00
	```
- A unique `orderId` is returned in the response.

**Business Rules Covered:**
- Shipping fee value sent when `hasShippingFee = false` is ignored.
- Total order price is calculated by the system without applying shipping fee.

**Status:**  
- Not Executed / Passed / Failed
