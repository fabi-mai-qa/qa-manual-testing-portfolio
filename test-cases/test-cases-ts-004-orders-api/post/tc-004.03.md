# TC-004.03 - Create Order with `hasShippingFee = true` and shipping fee value applied

## Description  
Validate successful order creation when `hasShippingFee` is set to true
and a valid shipping fee value is applied to the total price.

## Preconditions  
- User is authenticated with a valid API token.
- Seller exists with valid shipping configurations.
- Products exist and are linked to the respective seller.

## Endpoint
/orders  

## Payload
```json
{
  "userId": 123,
  "orderItems": [
    {
      "productId": 10,
      "quantity": 2,
      "unitPrice": 50.00,
      "sellerId": 145
    }
  ],
  "shippingBySeller": [
    {
      "sellerId": 145,
      "hasShippingFee": true,
      "shippingFeeValue": 10.00,
      "shippingType": "STANDARD",
      "estimatedDeliveryDays": 7
    }
  ]
}
```

**Expected Result:**  
- Order is created successfully.
- HTTP Status Code: 201 Created.
- Shipping fee is correctly applied.
- Total price is calculated as:
	```text
	 (2 × 50.00) + 10.00 = 110.00
	```
- A unique `orderId` is returned in the response.

**Business Rules Covered:**
- Shipping fee is applied only when `hasShippingFee = true`.
- `shippingFeeValue` must be greater than 0 when applied.
- Total order price is calculated by the system applying the shipping fee correctly.

**Status:**  
- Not Executed / Passed / Failed
