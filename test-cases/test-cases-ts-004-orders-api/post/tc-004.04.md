# TC-004.04 - Create Order with `hasShippingFee = true` and `shippingFeeValue = null`

## Description  
Validate that the system rejects order creation when `hasShippingFee` is set to true and `shippingFeeValue` is null.


## Preconditions  
- User is authenticated with a valid API token.
- Seller exists with valid shipping configurations.
- Products exist and are linked to the respective seller.

## Endpoint
/orders  

## Payload
```json
{
  "userId": 123,
  "orderItems": [
    {
      "productId": 10,
      "quantity": 2,
      "unitPrice": 50.00,
      "sellerId": 145
    }
  ],
  "shippingBySeller": [
    {
      "sellerId": 145,
      "hasShippingFee": true,
      "shippingFeeValue": null,
      "shippingType": "STANDARD",
      "estimatedDeliveryDays": 7
    }
  ]
}
```

**Expected Result:**  
- Order is not created.
- HTTP Status Code: 400 Bad Request.
- Response body returns a clear validation error message indicating that `shippingFeeValue` is mandatory when `hasShippingFee = true`.

**Business Rules Covered:**
- `shippingFeeValue` is mandatory when `hasShippingFee = true`.
- `shippingFeeValue` must be greater than 0 when applied.

- Not Executed / Passed / Failed
