# TC-004.06 - Create Order with multiple sellers and multiple products

## Description  
Validate successful order creation in a marketplace scenario with multiple sellers, multiple products and different shipping fee rules per seller.

## Preconditions  
- User is authenticated with a valid API token.
- Seller exists with valid shipping configurations.
- Product exists and is linked to the seller.

## Endpoint
/orders  

## Payload
```json
{
  "userId": 123,
  "orderItems": [
    {
      "productId": 10,
      "quantity": 2,
      "unitPrice": 50.00,
      "sellerId": 145
    },
    {
      "productId": 53,
      "quantity": 1,
      "unitPrice": 35.90,
      "sellerId": 210
    },
    {
      "productId": 27,
      "quantity": 1,
      "unitPrice": 119.50,
      "sellerId": 145
    }
  ],
  "shippingBySeller": [
    {
      "sellerId": 145,
      "hasShippingFee": false,
      "shippingType": "STANDARD",
      "estimatedDeliveryDays": 5
    },
    {
      "sellerId": 210,
      "hasShippingFee": true,
      "shippingFeeValue": 12.10,
      "shippingType": "EXPRESS",
      "estimatedDeliveryDays": 2
    }
  ]
}
```

**Expected Result:**  
- Order is created successfully.
- HTTP Status Code: 201 Created.
- Shipping fee is applied only for sellers where `hasShippingFee = true`.
- Total price is calculated as:
	```text
	 (2 × 50.00) + (1 x 35.90) + (1 x 119.50) + 12.10 = 267.50
	```
- A unique `orderId` is returned in the response.

**Business Rules Covered:**
- Each seller included in `orderItems` must have a corresponding entry in `shippingBySeller`.
- Shipping fee is applied only when `hasShippingFee = true`.
- Total order price is calculated as the sum of all items plus applicable shipping fees per seller.
- Marketplace supports multiple sellers and multiple products in a single order.
